# Workplace Update Prompt

Ты — архитектор рабочих пространств (workplace). Твоя задача — обновить существующий стандартизированный репозиторий (workplace) до новой версии бандла `workplace-scaffold`, следуя принципам безопасного слияния.

---

## Пререквизиты

Перед выполнением убедись:

1. Ты находишься в **корне существующего репозитория**, который был ранее инициализирован.
2. Директория `.data/workplace-init-bundle/` содержит новую версию шаблонов и скриптов (этот промпт находится внутри обновленного бандла). Если директория отсутствует — запроси у пользователя путь к новому бандлу.
3. У тебя есть доступ к инструментам: Bash, Read, Write, Edit, Glob, Grep.
4. Желательно, чтобы рабочее дерево Git было чистым (`git status` не показывает незакоммиченных изменений), чтобы можно было легко откатить слияние при ошибках.

---

## Фазы обновления

Обновление выполняется в 4 фазы. Каждая фаза завершается контрольной точкой с обратной связью от пользователя.

### Фаза 1 — Анализ текущего состояния (Discovery)

1. Прочитай текущие конфигурации и определи `project_code`, `project_type`, `language` (по содержимому `Makefile`, `README.md`, `.env.example`).
2. Проверь наличие кастомных изменений в системных файлах:

- Изменен ли `.gitignore` относительно стандартного?
- Есть ли пользовательские target-ы в `Makefile`?
- Какие специфичные инструкции добавлены в `AGENTS.md` (или `CLAUDE.md` / `GEMINI.md`)?

1. Сравни структуру `.data/workplace-init-bundle/standards/` с текущей директорией `knowledge/standards/`.

#### Контрольная точка 1

Выведи план обновления:

- Какие новые стандарты будут добавлены или обновлены.
- Какие скрипты будут перезаписаны.
- Потребуется ли ручное слияние (merge) для `AGENTS.md`, `README.md`, `Makefile` или `.gitignore`.

---

### Фаза 2 — Безопасное обновление (Safe Update)

В этой фазе файлы перезаписываются "как есть".

1. **Стандарты:** Скопируй все файлы из `.data/workplace-init-bundle/standards/` в `knowledge/standards/`. Существующие стандарты можно безопасно перезаписать.
2. **Скрипты:** Сгенерируй заново скрипты в `scripts/` на основе шаблонов из `.data/workplace-init-bundle/scripts/`, подставив найденный `project_code`. Перезапиши существующие системные скрипты.
3. **Вспомогательные файлы:** Обнови `.markdownlint.yml`, если он изменился в бандле.

#### Контрольная точка 2

Выведи список обновленных (перезаписанных) файлов. Запроси подтверждение перед переходом к слиянию шаблонов.

---

### Фаза 3 — Слияние шаблонов (Template Merge)

В этой фазе файлы **нельзя** просто перезаписывать. Нужно проанализировать разницу и "вмержить" новые практики из шаблонов в существующие файлы проекта:

1. **`AGENTS.md` (или `CLAUDE.md` / `GEMINI.md`):**
   - Сравни текущий файл с `templates/AGENTS-{project_type}.md.template`.
   - Если в шаблоне появились новые системные секции (например, новые проверки в Mandatory Checks) — добавь их в текущий файл.
   - **Строго сохраняй** все существующие специфичные для проекта инструкции пользователя.
2. **`Makefile`:**
   - Проверь `templates/Makefile-{project_type}.template`.
   - Добавь новые системные target-ы (например, связанные с бэкапами или линтингом).
   - **Строго сохраняй** пользовательские target-ы сборки.
3. **`.gitignore` / `.gitattributes`:** Добавь новые правила из шаблонов, если их не хватает в проекте.
4. **`README.md`:** Проверь, не изменилась ли рекомендуемая структура в `templates/README.md.template`. Если да, предложи точечные обновления.

#### Контрольная точка 3

Покажи `git diff` для измененных файлов (`AGENTS.md`, `Makefile`, и т.д.). Запроси ревью и подтверждение от пользователя.

---

### Фаза 4 — Валидация и финализация

1. Запусти `./scripts/workplace-bootstrap.sh` для проверки целостности структуры.
2. Выполни проверку `markdownlint-cli2 --config .markdownlint.yml README.md knowledge/**/*.md` (если инструмент доступен).
3. Убедись, что все индексы (`INDEX.md` в папке `knowledge/` и подпапках) актуальны.

#### Контрольная точка 4

Выведи финальную сводку:

```text
Workplace "${project_code}" успешно обновлен до новой версии стандартов.

Следующие шаги:
  1. Проверь изменения через git diff
  2. Закоммить обновление: git commit -m "chore: update workplace-scaffold to latest version"
```
